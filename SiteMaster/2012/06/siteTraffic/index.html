流量的两种切分方式

网站的流量是由 Visitors（访问者）行为导致。在 SiteMaster 中，是通过基于 Cookie 技术的 指纹算法，尽可能识别准确的“自然人”。Visitors 会多次 Visits（访问）网站，每次 Visit 都会造成多个 PageViews （页面浏览）。

当把网站的所有的 Visits 作为一个大的集合时，每一个 Visit 就是这个集合里的个体。SiteMaster 有两种切分方式将这个集合像蛋糕一样进行完整切分，使：每一个子集间无交集，且所有子集的并集为原集合。

根据 Visits 的 Cookie 历史

V 代表所有 Visits 的集合。

A 代表所有 有与此 SiteID 关联项目广告历史的 Visits。

B 代表所有 无与此 SiteID 关联项目广告历史的 Visits。

此时 A 与 B 两集合无交集，A 与 B 并集为 V。

注意几个基本概念：

* 广告历史

Visitors 在造成 Visit 是，会携带其 AdMaster Cookie，包含了其与 AdMaster 广告交互（曝光或点击）的历史信息。SiteMaster 只追溯最近 30 天的广告历史信息。

* 关联项目

指在 TrackMaster 系统项目属性中，关联了此 SiteID 的项目，项目ID 与 SiteID 是多对一的关系

* 关联项目广告历史

存在 Visitors 有广告历史，但这些历史的所属 项目ID 只有部分，甚至没有与 SiteID 关联。

SiteMaster 广告模块的报告，分析的数据集合为 A，即仅对有关联项目广告历史的 Visits 进行统计分析。

根据 Visits 的 Landing Page URL 与 Referral URL

V 代表所有 Visits 的集合。
X 代表所有 Landing Page 带有合法 smt 参数的 Visits。
Y 代表所有 Landing Page 不带有合法 smt 参数的 Visits。

此时 X 与 Y 两集合无交集，X 与 Y 并集为 V。

在集合 Y 中的 Visits，再进行切分：

  • Y1 代表 Referral URL 的 Hostname 在搜索引擎域名列表中的 Visits。
  • Y2 代表 Referral URL 的 Hostname 在社会化媒体域名列表中的 Visits。
  • Y3 代表 Referral URL 的 Hostname 在不在搜索引擎与社会化媒体列表的 Visits。
  • Y4 代表 Referral URL 为空的 Visits。

此时 Y1 Y2 Y3 Y4 四集合无交集，Y1 Y2 Y3 Y4 并集为 Y。
此时 Y1 Y2 Y3 Y4 X 五集合无交集，Y1 Y2 Y3 Y4 X 并集为 V。

注意几个基本概念：

  • smt 参数

    smt 参数包括 项目、媒体、广告位、创意、关键字 五个。有一定的合法条件，只有合法的 smt 参数组合才被记录

    * 关联项目

    smt_b 参数扩充时，不考虑关联关系，但在数据展现时，ID 是否解释成具体名称需要考虑关联关系。
